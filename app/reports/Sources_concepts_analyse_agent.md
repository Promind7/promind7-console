## 2. Analyse par fichier

### 2.1 `Critical_Thinking_Bilingue_ar.docx`

**Contenu** : plaquette pédagogique (source indiquée : *Middletown Thrall Library* — tutoriel « critical thinking »).  
**Axes** : toute information peut être incomplète ou fausse ; **désinformation** vs erreurs innocentes ; liste de **questions** à se poser (qualification de la source, actualité, biais, propagande, mobile, vérité partielle, sources multiples) ; recherche en ligne = risque ; intérêt de **plusieurs sources** et de l’aide **documentaire** (librarian).

**Forces pour proM7**  
- Aligné **lucidité** et **anti-bulle** (pas de promesse magique).  
- Pont naturel avec **vérifier les offres**, **LinkedIn**, **IA** (M6), **chiffres du marché** (M1).

**Limites / précautions**  
- **Arabe classique** dans le doc : le script oral parcours reste en **darija** — à reformuler, pas copier-coller l’arabe du fichier.  
- **URLs US** (`thrall.org`, base *Opposing Viewpoints*) : **ne pas** les mettelles dans le script Maroc telles quelles ; remplacer par une formulation générique (« plusieurs sources », « sites officiels », « documentation à jour »).  
- Question EN du fichier : *« Does the information … satisfy my needs? If so, why not? »* — probable coquille (**If not**). À corriger si on réutilise la checklist.

**Dispatch cible (indicatif)**  
- **M1 L10–L16** : lire offres, médias, « rumeurs » sur le marché — une **mini-checklist** (3–5 questions) en darija.  
- **M6 L51–L53** : info numérique, IA, sources — **renforcer** l’idée « vérifier avant d’agir ».  
- **M2** (option) : décisions en entreprise — bref rappel « info ≠ vérité complète » (sans alourdir).

### 2.2 `Entretien_Darija_Seulement.docx`

**Contenu** : ~10 questions types (weaknesses, défendre une décision, dilemme éthique, client difficile, deadline manquée, pourquoi quitter le poste, différenciation, mauvaise nouvelle, délégation, **salaire**, stress, conflit collègues). Chaque fois : **intention du recruteur** + **structure de réponse** en darija (STAR implicite, honnêteté, exemples).

**Forces**  
- Déjà **aligné tonalité** parcours (darija + logique recruteur).  
- Très **actionnable** pour scripts **entretien / oral pro**.

**Limites**  
- Questions en **anglais** : à relier explicitement aux leçons **M7** (pratique EN) et **M5** (emploi).  
- **Salaire** : garder prudence proM7 (pas de fourchettes inventées ; méthode > chiffre).

**Dispatch cible**  
- **M5** (leçons entretien / psychologie recherche emploi / CV selon où tu places l’oral aujourd’hui).  
- **M7 L60–L61** : même matière **côté formulation EN** (garder question EN + explication darija).  
- **M4** : sous-ensemble **stage** (éthique, deadline, conflit, client — adapter « work » → « stage / projet »).  
- **M2 L27** (recrutement vu de l’intérieur) : 1–2 questions pour montrer **l’autre côté de la table**.

### 2.3 `Negotiation_Bilingue_darija.docx`

**Contenu** :  
- 6 lignes type **séparer personnes / problème**, **intérêts vs positions**, émotions, reconnaissance, message positif, sortir du cycle action-réaction.  
- **Règle 7/38/55** (Mehrabian, souvent **surrésumée** en formation).  
- **ZOPA** (zone d’accord possible).  
- 6 conseils (besoins humains, réserve d’info, adaptation émotion/logique, pause, motivations, temps / deadlines).

**Forces**  
- Vocabulaire **FR/EN** utile (ZOPA, intérêts) pour **négociation salaire / stage**.  
- Darija déjà présente pour titres.

**Limites / précautions**  
- Attribution **« Harvard »** : le public peut associer à *Getting to Yes* — si tu gardes le label, **nuancer** (principes inspirés de la littérature de négociation, pas cours juridique).  
- **7/38/55** : en pédagogie sérieuse, c’est une **heuristique**, pas une loi exacte ; ajouter une phrase de prudence pour ne pas contredire la science de la communication.  
- Liste « 6 besoins » (certitude, incertitude, etc.) : mélange **non académique strict** — utile comme **accroche**, pas comme théorie officielle.

**Dispatch cible**  
- **M5** (négociation, préparation entretien, salaire avec prudence).  
- **M4** (convention de stage, discussion avec tuteur — version adoucie).  
- **M3** (gestion émotions / conflit) : points « personnes vs problème » et émotions.  
- **M7** : si tu traites **pitch / reformulation** — bribes, pas tout le pack.

**Forces**  
- Très **compatible** oral vidéo proM7 et **M7** (présentation).  
- Cohérent avec l’exercice « vidéo 1 min » évoqué dans `reports/Sources_dispatch_en_attente.md` (V5).

**Limites**  
- Risque de **chevauchement** si on duplique trop avec **M7 L62** (simulation) — **répartir** (L62 = pratique ; ce doc = **principes**).

**Dispatch cible**

- **M5** : oral d’entretien, pitch court.  
- **M3** : prise de parole en réunion (aperçu).
